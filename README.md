# Quaid Evals

Automated benchmark suite for [Quaid](https://quaid.app) - the persistent memory system for AI agents.

Runs DAB, gbrain-evals, and BEAM benchmarks on every release. Publishes results to [GitHub Pages](https://quaid-app.github.io/quaid-evals).

## Benchmarks

| Benchmark | What it tests | Reference |
|-----------|--------------|-----------|
| **DAB** (Doug Aillm Benchmark) | Full retrieval quality on MSMARCO corpus: FTS, semantic, MCP, performance, integrity, collections | Custom - 215 pts max |
| **gbrain-evals** | P@5 / R@5 on MSMARCO dev queries (500 queries, real qrels - industry standard) | GBrain: P@5=49.1%, R@5=97.9% |
| **LoCoMo** | Multi-session conversational memory recall (~300 questions) | Mem0: 91.6% |
| **BEAM** | Memory at extreme scale (100K / 1M / 10M tokens) | Mem0: 64.1% @ 1M, 48.6% @ 10M |

## Run Locally

### Prerequisites
- Quaid installed (`quaid --version`)
- Python 3.11+
- Node.js 20+

### Setup

```bash
git clone https://github.com/quaid-app/quaid-evals
cd quaid-evals

# Install Quaid (if not already installed)
bash scripts/install-quaid.sh

# Set up corpus
bash scripts/setup-corpus.sh

# Install Python deps
pip install -r scripts/requirements.txt
```

### Run benchmarks

```bash
# DAB only
bash benchmarks/dab/run.sh

# gbrain-evals adapter
bash benchmarks/gbrain-evals/run.sh

# Both + generate report
bash benchmarks/dab/run.sh
bash benchmarks/gbrain-evals/run.sh
python scripts/generate-report.py
```

### Build the site

```bash
cd site
npm install
npm run build    # outputs to site/dist/
npm run dev      # local preview
```

## Interpreting Scores

**DAB** - Higher is better. 185/215 = 86% is our current baseline (Quaid v0.9.8).

**P@5 (Precision at 5)** - Of 5 results returned, how many are relevant? GBrain achieves 49.1%. Higher = less noise.

**R@5 (Recall at 5)** - Of all relevant pages, how many appear in the top 5? GBrain achieves 97.9%. Higher = nothing slips through.

**BEAM** - Percentage score at each scale tier. Tests memory systems where context stuffing is impossible.

## Architecture

Results are stored as JSON files in `results/` and committed to the repo. The Astro site reads `site/src/data/report.json` (generated by `scripts/generate-report.py`) and renders the history chart and comparison table statically - no backend needed.

## CI/CD

- **Manual**: Run any benchmark on any version via GitHub Actions → `Run Benchmarks (Manual)`
- **On release**: Quaid's release workflow can trigger `repository_dispatch` with `event-type: quaid-released` and `client_payload.version: v1.2.3`
